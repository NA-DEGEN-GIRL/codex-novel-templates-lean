# Dialogue Voice Log

> 캐릭터 보이스 drift 방지 + 역할 고착 감지용 운용 로그.
> `settings/03-characters.md`의 앵커가 원천이다. 이 파일은 대체하지 않는다.
> 원문 대사를 통째로 저장하지 않는다.

## 행 유형

이 로그에는 두 종류의 행이 있다.

- **이탈 행**: 앵커 대비 보이스 변화가 있을 때. 모든 열을 채운다.
- **role-only 행**: 보이스는 앵커와 동일하지만 대화 기능만 기록. 톤 델타/관계톤/지향은 `—`로 채운다.

## 기록 기준

- **대화 기능(1열)은 매화 등장 시 항상 기록한다** — 역할 고착 판정의 근거.
- **톤 델타/관계톤/지향은 앵커에서 이탈한 경우만 기록한다** — 보이스 drift 방지의 근거.
- **주요 캐릭터**: 의미 있는 대사 1회 이상이면 기록
- **조연**: 3회 이상 대화 교환 + 어투 변화/관계 변화/재등장 복선 중 하나일 때만 기록
- **에피소드당 최대 5명**. 초과 시 이탈 행 우선, role-only 행은 후순위

## 대화 기능 태그 (고정 어휘)

추궁자 / 회피자 / 중재자 / 폭로자 / 도전자 / 목격자 / 완충자

## Pruning 규칙

- **role-only 행**: 캐릭터별 최근 5행만 유지
- **이탈 행**: 최근 3화 + 마지막 이탈 1건 유지
- **승격**: 동일 패턴이 3화 이상 반복 → `03-characters.md`로 승격 검토. 승격 후 관련 이탈 행은 1행으로 압축
- **파일 하드캡**: 150행. 초과 시 오래된 role-only 행부터 제거
