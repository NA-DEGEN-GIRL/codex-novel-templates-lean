# 캐릭터 정보 보유 현황 (Knowledge Map)

> 각 캐릭터가 **현재 알고 있는 정보**와 **습득 시점**을 추적한다.
> "아직 모르는 것을 아는 것처럼 말하는" 모순을 방지하는 핵심 파일이다.

---

## 사용법

- 집필 전: 이번 화에 등장하는 캐릭터들이 **현재 무엇을 알고 무엇을 모르는지** 확인
- 집필 후: 캐릭터가 새 정보를 습득했으면 갱신
- **대화 작성 시**: 화자가 해당 정보를 알고 있는지 이 파일로 반드시 확인

---

## 정보 카테고리

소설에 맞게 카테고리를 추가/수정한다. 아래는 예시:

| 카테고리 | 예시 |
|---------|------|
| 정체/비밀 | "A의 진짜 정체는 ~이다" |
| 사건/음모 | "~가 ~를 배신했다" |
| 능력/기술 | "A는 ~라는 기술을 가지고 있다" |
| 장소/세력 | "~라는 곳이 존재한다" |
| 관계 | "A와 B는 실은 ~관계다" |

## 갱신 규칙

### 추가 시점
아래 상황이 본문에 등장하면 **반드시** 갱신한다:
- 캐릭터가 **새로운 사실을 알게 됨** (직접 목격, 전해 들음, 추론, 발견)
- 캐릭터가 **다른 캐릭터에게 정보를 전달**함
- 캐릭터가 **오해**를 하게 됨 (잘못된 정보 습득)
- 비밀이 **폭로**됨

### 주의사항
- **"알고 있음"은 본문에 근거가 있어야 한다.** 장면에서 직접 보거나, 누군가에게 들었거나, 합리적으로 추론한 경우만.
- 같은 공간에 있었다고 자동으로 모든 정보를 아는 것은 아니다. **대화/목격 장면이 본문에 있어야** 한다.
- "부분적으로 앎(△)"도 기록한다. 예: "A가 수상하다는 것은 알지만 정체까지는 모름"

### 검증 시점
- **unified-reviewer (continuity 모드)**: 캐릭터가 아직 모르는 정보를 언급하거나 반응하지 않는지 검증
- **집필 전**: 등장 캐릭터들의 현재 정보 보유 상태를 확인하고, 대화에서 정보 갭이 자연스럽게 반영되는지 확인
